LP Comment Research Parser

Private internal research and analytics tool for collecting public YouTube comments from selected technical channels and videos through YouTube Data API v3.

The API Client is used internally for topic analysis, user problem clustering, technical discussion analysis, product research, and market research in 3D printing and additive manufacturing.

API Client status

  • Access model: private/internal, not public
  • Authentication: API key only, no OAuth
  • API scope: read-only public data
  • Data processing: offline/local analysis
  • Output: JSONL/CSV/XLSX/Markdown reports
What the tool collectsPublic video metadata, public top-level comments, public replies, timestamps, like counts, video IDs, comment IDs, and parent-child relationships.
What the tool does not doIt does not upload, modify, delete, moderate, rate, or publish YouTube content. It does not access private user data.
How the data is usedOffline topic clustering, recurring problem analysis, technical request extraction, and internal research reporting.
Reliability safeguardsCheckpoint/resume behavior, incremental local writes, duplicate removal, and per-video/per-page error logs.

Read-only YouTube Data API endpoints

EndpointPurpose
youtube.channels.listResolve a public channel and retrieve channel metadata.
youtube.playlistItems.listRetrieve public videos from the channel uploads playlist.
youtube.commentThreads.listRetrieve public top-level comment threads for selected videos.
youtube.comments.listRetrieve public replies inside selected comment threads.

Quota extension rationale

Complete channel-level research may require crawling hundreds of videos. Each comments page and each replies page is paginated, and active technical videos often contain many replies. The default daily quota is not sufficient for full public comment and replies collection for a complete technical channel.

Requested daily quota: 100,000 units/dayPeak rate target: moderate / non-aggressiveUse case: internal analytics and research
