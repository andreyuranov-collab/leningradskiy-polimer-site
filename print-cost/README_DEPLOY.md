# LP_PrintCostWebApp_v00.01_test

Тестовая веб-версия калькулятора стоимости 3D-печати для сайта НПО «ЛЕНИНГРАДСКИЙ ПОЛИМЕР».

Источник логики: `LP_PrintCostApp_v00.15_test`.

## Что входит

- `index.html` — страница калькулятора.
- `assets/app.js` — клиентская логика парсинга G-code и расчёта стоимости.
- `assets/styles.css` — оформление под технический стиль бренда.
- `assets/lp_logo.png` — логотип из исходного приложения.
- `data/defaults.json` — станки, материалы и правила расчёта, выгруженные из `lp_print_cost.db`.

## Что умеет v00.01_test

- Расчёт выполняется полностью в браузере.
- Файл печати не отправляется на сервер.
- Поддерживается прямой парсинг `.gcode`, `.gco`, `.gc`, `.txt`.
- `.3mf` поддерживается частично: если внутри архива есть G-code, он будет прочитан.
- `.stl` не рассчитывается автоматически, потому что STL не содержит времени печати, поддержек, веса расхода и настроек слайсера. Для STL нужно сначала получить G-code в слайсере или заполнить параметры вручную.
- Перенесены формулы из `calculator.py`: материалы, электричество, амортизация, сушка, адгезия, износ сопла, оператор, минимальный заказ, НДС.
- Перенесены основные признаки из `gcode_parser.py`: слайсер, принтер, время, материалы, поддержки, brim, raft, prime/wipe tower, purge/flush, наличие миниатюры.

## Важное ограничение публичной версии

Это статический сайт. Все коэффициенты, цены материалов, ставки оператора и коммерческий коэффициент лежат в `data/defaults.json` и будут видны пользователю в браузере.

Если эти данные нельзя показывать публично, используйте веб-версию только как внутренний инструмент или перенесите расчёт на сервер/API.

## Размещение на Cloudflare Pages + GitHub

### Вариант А. Отдельная страница `/print-cost/`

1. Распакуйте архив `LP_PrintCostWebApp_v00.01_test.zip`.
2. В репозитории сайта создайте папку:

```text
print-cost/
```

3. Скопируйте внутрь папки содержимое архива:

```text
print-cost/index.html
print-cost/assets/app.js
print-cost/assets/styles.css
print-cost/assets/lp_logo.png
print-cost/data/defaults.json
```

4. Добавьте ссылку на калькулятор в меню или на главную страницу сайта:

```html
<a href="/print-cost/">Калькулятор стоимости печати</a>
```

5. Загрузите изменения в GitHub:
   - `Add file` → `Upload files`, если работаете через браузер;
   - или обычный `git add .`, `git commit`, `git push`, если работаете через Git.

6. Cloudflare Pages сам пересоберёт сайт после обновления репозитория.

7. Проверьте адрес:

```text
https://leningradskiy-polimer.pages.dev/print-cost/
```

### Вариант Б. Сделать калькулятор главной страницей

Скопируйте содержимое архива прямо в корень репозитория сайта. Тогда `index.html` калькулятора станет главной страницей сайта.

Этот вариант не рекомендуется, если на сайте уже есть главная страница компании.

## Проверка после размещения

1. Откройте страницу калькулятора.
2. Загрузите тестовый `.gcode`.
3. Проверьте, что подтянулись:
   - время печати;
   - расход материала;
   - материал/материалы;
   - признаки поддержек, brim/raft, башни и purge/flush;
   - итоговая цена и цена за деталь.
4. Измените количество деталей и деталей на пластине — итог должен пересчитаться сразу.

## Что стоит сделать в следующей версии

- Добавить полноценный серверный режим, если коммерческие коэффициенты нельзя раскрывать публично.
- Добавить отдельный блок технологических отходов: supports, prime/wipe tower, purge/flush.
- Добавить сохранение заявки/расчёта в PDF или отправку на e-mail через серверную функцию.
- Добавить админ-файл для обновления материалов и станков без правки JSON вручную.
- Добавить интеграцию с формой заявки на сайте.
